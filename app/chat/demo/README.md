# iNGEN Chat Demo Presenter Notes

Route: `/chat/demo`

Control: press `Space` or click `Next / Space` to advance one step. `Back` and `Reset` are available in the presenter control panel.

Target runtime: 4 minutes. With 15 steps, pace each step at roughly 15-18 seconds and pause only on the results, compare, assessment, pipeline, and analytics surfaces.

## Script

1. Aristotle Greeting
   Say: "This is the recruiter command center. Aristotle is not a chatbot bolted onto ATS data; it is an operating layer over evidence, trust, and workflow."

2. Search Command
   Say: "The recruiter starts with intent, not Boolean syntax: senior Rust developers, backend systems proof, and strong GitHub signal."

3. Reasoning Stream
   Say: "Before results appear, the system exposes its reasoning path: parse requirements, scan proof, score GitHub signal, check trust metadata, then rank."

4. Ranked Results
   Say: "The rail opens with seven candidates. Each result is tied to proof bundles, readiness, trust level, and explainable ranking signals."

5. Compare Command
   Say: "The recruiter can pivot without leaving the workspace. The command becomes an operating action, not a separate report."

6. Recommendation
   Say: "The compare view shows why one candidate is recommended and where the risks remain. This is designed for accountable human decision-making."

7. Save Command
   Say: "The recruiter saves the result set as a reusable Rust Backend shortlist."

8. Pool Created
   Say: "The pool becomes a living record: candidates, filters, auto-refresh rules, and role context are preserved."

9. Simulation Command
   Say: "The next step launches a role-realistic assessment from the same pool."

10. Assessment Launcher
   Say: "The launcher is pre-filled with the pool, role rubric, duration, integrity settings, and candidate-facing copy. The recruiter can confirm rather than rebuild the workflow."

11. Pipeline Command
   Say: "Aristotle can queue conditional workflow actions, such as moving Alex and Priya only after they pass the simulation."

12. Conditional Pipeline
   Say: "The pipeline board shows the action as queued, not blindly executed. Conditions and audit context remain visible."

13. Analytics Command
   Say: "The recruiter asks for funnel health scoped to the Rust Backend role."

14. Funnel Health
   Say: "Analytics connects sourcing, assessment completion, trust distribution, pool health, and pathway effectiveness in one role-scoped snapshot."

15. End Card
   Say: "iNGEN is proof-first hiring: a system of record for software delivery truth."

## IBM-Specific Talking Points

- Enterprise scale: The demo frames search over thousands of proof bundles while keeping recruiter workflow fast and auditable.
- Trust layer: Every recommendation is grounded in verification level, evidence freshness, integrity flags, and proof artifacts.
- Audit trail: Commands, reasoning steps, saved pools, assessment launches, and pipeline moves become reviewable operating records.
- Explainability: Aristotle exposes why candidates rank, why one is recommended, and what risks still require human review.
- Company-authored pathways: Role rubrics and pathways let enterprise teams define what evidence matters for their own engineering standards.
- Governance fit: Conditional actions demonstrate automation with controls instead of opaque auto-hiring decisions.
- Integration story: The command center can sit above ATS, assessment, GitHub, and internal pathway systems as an orchestration layer.
